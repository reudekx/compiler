[언어 기능]

데코레이터, 파이프라인

데코레이터로 파이프라인 역할을 수행함.


익명 함수로 데코레이터를 생성할 수 있는가?

데코레이터의 정의 자체를 지연 평가를 할지 명확히 정해야 한다.

지연 평가의 대상에는 변수의 사용이 포함될 수 있을까?

가령,

fun(k) return x + k end

위와 같은 함수는 자신의 맥락에 x라는 외부의 변수를 캡쳐하게 될 것이다.

즉, x라는 변수가 여러 개 존재할 수 있음에도 최초에 캡쳐한 scope의 x만을 가리키게 될 것이다.


만약 지역 변수의 선언 시 항상 let을 사용한다고 가정해보자.

만약 let x 와 같은 구문이 같은 scope에서 2번 등장한다면?

그 사이에 위 함수의 선언이 존재한다면 함수는 첫번째 x를 캡쳐하게 될 것이다.


기명 함수의 경우는 어떤가?

전방 선언 없이 함수를 호출해도 된다고 생각해보자.

이는 자연스러운 동작이라고 볼 수 있을까?

반대로 함수의 선언이 앞서 작성되고, 캡쳐한 변수가 후에 정의되는 경우를 생각해보자.

가령,

fun g() return x end

let x = 1

print(g())

위의 코드에서 g 함수는 어떤 x를 캡쳐한 것인가?


[스코핑]

결과적으로 편의성과 단순성(일관성?)을 모두 충족하는 규칙을 정의하는 것은 어려워 보인다.

특히 익명 함수와 기명 함수를 다르게 처리해야할지 아니면 동일하게 처리할 수 있는지 고민을 해봐야 한다.


가령 한 익명 함수를 정의하고 동시에 변수에 저장을 했을 때, 해당 변수의 이름을 이용하여 이 익명 함수를 호출할 수 있을까?

let f = fun(k)
    if k == 1 then
        return 1
    else
        return f(k - 1) * k
    end
end

일단 파이썬에서는 가능하다.

f = lambda x: 1 if x == 1 else f(x - 1) * x

(위와 같은 코드가 유효함)


그러나 익명 함수의 경우 해당 선언문이 실행되기 전에는 호출할 수 없다. (해당 변수가 아직 존재하지 않을테니 말이다.)

반면 기명 함수는 정의가 코드의 나중 부분에 작성되어 있더라도 호출할 수 있다. (hoisting)

대부분의 언어에서 class와 function definition과 같은 '할당'이 아닌 '정의'인 경우 해당 statement가 scope의 맨 위로 끌어올려지는 것으로 처리되는 것 같다.


함수 자체의 hoisting말고도 주목해야 하는 특성이 한 가지 더 존재한다.

앞서도 언급했었지만, 바로 아직 선언되지 않은 변수를 캡쳐할 수 있다는 점이다.

파이썬의 경우를 살펴보자.

def scope():
    f = lambda: print(x)
    x = 10
    f()

scope()

위의 코드는 유효하며 잘 실행된다.

이에 대해선 어떻게 생각해야 할까?

lua의 경우도 살펴보자.

function scope()
    f = function() print(x) end 
    local x = 10
    f()
end

scope()

위의 코드는 실행되지만, 익명 함수는 전역 함수 x를 캡쳐하고, 해당 변수의 값이 할당되지 않았기 때문에 nil이 출력된다.
이는 익명/기명 함수 모두에 해당하며, hoisting 또한 발생하지 않는다.


따라서 lua에서 순환 참조 등을 이용하고 싶다면, table 객체를 통해 우회적(명시적이라고 해야할지,)으로 접근하면 된다.

가령 self 라는 인자를 설정하여 self 객체의 메서드를 호출하는 것이다.

언제든지 self에 해당 메서드가 추가된다면 무리없이 호출될 것이다.




만약 이런 우회적인 방법 없이, 명시적으로 변수를 예약하듯이 캡쳐할 수 있을까?

가령 scope의 layer를 나타내는 예약어를 추가하면 어떨까?

클로져가 특정 변수를 캡쳐하기 위해서는, 해당 변수가 나타날 때까지 scope의 nvironment를 거슬러 올라가야 한다.

이때 원하는 위치까지만 올라가고, 해당 environment에 변수가 존재하지 않는다면 빈 변수를 생성하고 그 참조를 보관하도록 하는 것은 어떨까?

다만 이 경우 let을 통해 같은 이름의 변수가 다시 선언된다면 이 변수를 캡쳐할 것인지, 아니면 해당 변수는 별개의 것으로 간주해야 할지 모호해진다는 문제가 생긴다.


그냥 lua의 스타일 대로 해결하는 것이 가장 괜찮아 보인다.


[문법]

코드 scope를 어떻게 구분할 건지?

    1. end 사용
    2. 중괄호 { } 사용

    중괄호를 예시로 함수를 작성해보자

    let scope = fun() {
        let f = fun() { print(x) }
        let x = 10
        f()
    }

    가독성이 썩 좋아보이진 않는다. 그냥 end가 가장 무난해보이는데, end 말고 다른 단어를 쓸 수도 있을 것 같아. 가령 세미콜론은 어떤가?

    let scope = fun()
        let f = fun() print(x);
        let x = 10
        f()
    ;

    하지만 세미콜론이 중첩되는 경우라면?


    let scope = fun() let f = fun() print(x);;

    let scope = fun() let f = fun() print(x) end end



가령 기명 함수를 없앤다면? fun이라는 예약어도 없애면?

    일단 매개변수를 선언할 수 있어야 하니 다음과 같이 작성해보자.

    let person = [name, age]
        let f = [] print(x) end
        let x = 10
        f()
    end

    위의 경우 매개변수가 없는 함수인 경우가 애매하다.

    let person = fun(name, age)
        let f = fun() print(x) end
        let x = 10
        f()
    end

    fun으로 처리하고 익명함수만 제공해도 될 것 같다.

    if 조건1 then

가령 표현식 지향 언어를 만들고 싶다면?

모든 코드 블록은 값을 반환하게 될 것이다. (반환할만한 적절한 값이 없더라도, 적어도 void나 null을 반환할 것이다.)

이때는 코드 블록의 시작과 끝, 또한 표현식의 끝을 나타내는 것이 매우 중요할 것이다.

중괄호와 세미콜론의 존재가 필연적인 것 같다.

iterable한 객체에서 원소를 하나식 꺼내 변수에 저장하는 for문이 있다.

while문은 위에서 정한 if와 유사하게 표현할 수 있다.

while (조건)

end

그렇다면 for문은 어떤가?

for (a, b, c in list)

end

위의 형태를 따르도록 강제하자. 일관성 있게 괄호를 쓰는 게 좋아보인다.

[종료, 반환 키워드]


이번엔 반복문의 진행과 종료를 위한 키워드를 생각해보자.

continue, break

또한 함수의 탈출과 동시에 값을 반환하는 return 키워드도 있다.

파이썬에서의 것과 동일하게 작동하는 것으로 정하자.


모든 것을 합쳐서 예시 코드를 작성해보자.


let example = fun(k)
    if (k < 0)
        k = -k
    elseif (k == 0)
        return 0
    end

    while (true)
        k += 1
        if (k == 10) break
    end

    let sum = 0

    for (i in range(k))
        sum += i
    end

    return sum
end
