[문법 정의]

대략적으로 작성함.
표기법이 잘못 되었거나 논리적으로도 맞지 않은 부분 존재함.


수정 1) block을 항상 'ok'로 끝나는 것으로 수정
    -> 하려고 했으나 조건문에서처럼 꼭 'ok'로 끝나지 않기도 함.
        -> 조건문 블록을 따로 처리.

각 논터미널은 맥락이 중요하지 않음.
어차피 논터미널이 확장되는 시점에 필요한 Bytecode 연산이 적절히 스택에 추가될 것이기 때문
    -> 즉 이후 자식 논터미널들을 파싱할 때는 생성되었을 때의 맥락이 전혀 중요하지 않음.
    -> 그래서 elseBlock을 따로 정의할 필요없이 그냥 block을 이용하면 됨.

==================================================================================

program
    -> block

programBlock
    -> expresion programBlock
    | EOF

block
    -> expression block
    | 'ok'

ifBlock
    -> expression ifBlock
    | 'elif' elifBlock
    | 'ok'

elifBlock
    -> expression elifBlock
    | 'else' block
    | 'ok'

expression
    -> variableDefinition
    | assignment
    | functionDefinition
    | call
    | loop
    | conditional
    | jump
    | identifier
    | dotNotation
    | literal
    | 'do' block 'ok'

variableDefinition
    -> 'let' identifier '=' expression

assignment
    -> chainedIdentifier '=' expression

functionDefinition
    -> 'def' '(' { parameter } block 'ok'

call
    -> chainedIdentifier '(' { argument }

loop
    -> whileLoop
    | forLoop

whileLoop
    -> 'while' expression 'do' block 'ok'

forLoop
    -> 'for' parameter 'in' expression 'do' block 'ok'

conditional
    -> 'if' expression 'do' ifBlock

jump
    -> 'return' expression?
    -> 'break' expression?
    -> 'continue'

parameter
    -> chainedIdentifier { ',' parameter }
    | ')'

argument
    -> expression { ',' argument }
    | ')'

chainedIdentifier
    -> expression { '.' chainedIdentifier }
    | identifier

chainedIdentifier

literal
    -> number
    | string
    | 'true'
    | 'false'

number
    ->

string
